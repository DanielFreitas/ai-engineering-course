# Redis e performance com cache

## Objetivo

Ao final deste módulo você consegue adicionar cache Redis a uma chamada de LLM — definindo chave determinística, TTL e estratégia de invalidação — e medir o impacto em latência e custo com e sem cache.

---

## Por que cache é especialmente valioso em sistemas de IA

LLMs têm dois custos que o cache ataca diretamente:

| Custo | Sem cache | Com cache (hit) |
|---|---|---|
| Latência | 1–5 segundos típicos | `<` 5ms (Redis) |
| Custo monetário | Tokens por requisição | Zero |

O problema: nem toda saída de LLM é reutilizável. Respostas que variam por usuário, por data ou por dados dinâmicos não devem ser cacheadas globalmente. A habilidade essencial deste módulo é **identificar o que é seguro cachear e por quanto tempo**.

---

## Tipos de cache em sistemas de IA

| Tipo | O que guarda | Quando usar |
|---|---|---|
| **Cache de resposta** | Resposta completa para um prompt exato | FAQs, análises de texto estático |
| **Cache de contexto de sessão** | Histórico ativo da conversa | Acesso rápido entre turnos |
| **Cache de embedding** | Vetor calculado para um texto | Documentos que não mudam |
| **Cache semântico** | Resposta para perguntas semanticamente similares | Perguntas frequentes com reformulação |

---

## Convenção de chaves Redis

Todas as chaves seguem o padrão: `{namespace}:{tipo}:{identificador}`

```
ai:response:{sha256_do_prompt}          # resposta de LLM
ai:session:{session_id}                 # contexto de sessão
ai:embed:{document_id}:{chunk_index}    # embedding calculado
ai:ratelimit:{user_id}:{janela_hora}    # rate limiting
```

Incluir **versão do prompt** na chave garante invalidação automática quando o prompt muda:

```
ai:response:{prompt_version}:{sha256_do_prompt}
# ex: ai:response:v1.0.0:8a3f2c...
```

## Cache de resposta de LLM

```python
# cache/llm_response_cache.py
import hashlib
import json
from datetime import datetime, timezone
import redis.asyncio as redis

class LLMResponseCache:
    def __init__(self, redis_client: redis.Redis, prompt_version: str):
        self._redis = redis_client
        self._prompt_version = prompt_version

    def _make_key(self, prompt: str, model: str) -> str:
        """Gera chave determinística para o par (prompt, modelo, versão)."""
        normalized = prompt.strip().lower()
        content = f"{model}:{self._prompt_version}:{normalized}"
        digest = hashlib.sha256(content.encode()).hexdigest()
        return f"ai:response:{self._prompt_version}:{digest}"

    async def get(self, prompt: str, model: str) -> dict | None:
        """Retorna resposta cacheada ou None se não encontrada."""
        key = self._make_key(prompt, model)
        raw = await self._redis.get(key)
        if raw:
            return json.loads(raw)
        return None

    async def set(
        self, prompt: str, model: str, response: dict, ttl_seconds: int
    ) -> None:
        """Armazena resposta no cache com TTL especificado."""
        key = self._make_key(prompt, model)
        payload = {
            **response,
            "cached_at": datetime.now(timezone.utc).isoformat(),
            "ttl_seconds": ttl_seconds,
        }
        await self._redis.set(key, json.dumps(payload), ex=ttl_seconds)
```

### Integrando ao serviço

```python
# services/cached_analysis_service.py
import time
import logging

logger = logging.getLogger(__name__)

class CachedAnalysisService:
    def __init__(self, llm_client, cache: LLMResponseCache):
        self._llm = llm_client
        self._cache = cache

    async def classify_text(self, text: str, model: str = "gpt-4o") -> dict:
        start = time.monotonic()

        # Tenta buscar do cache
        cached = await self._cache.get(text, model)
        if cached:
            logger.info("cache_hit", extra={"model": model})
            cached["from_cache"] = True
            cached["latency_ms"] = int((time.monotonic() - start) * 1000)
            return cached

        # Cache miss — chama o modelo
        logger.info("cache_miss", extra={"model": model})
        raw, usage = await self._llm.complete(
            system="Classifique o sentimento como positive/negative/neutral. Retorne JSON.",
            user=text,
        )

        result = {
            "classification": raw,
            "usage": usage,
            "from_cache": False,
            "latency_ms": int((time.monotonic() - start) * 1000),
        }

        # Armazena no cache — TTL de 1 hora para textos estáticos
        await self._cache.set(text, model, result, ttl_seconds=3600)
        return result
```

## TTLs por tipo de conteúdo

| Tipo de conteúdo | TTL recomendado | Justificativa |
|---|---|---|
| FAQ / perguntas frequentes | 24h | Muda raramente |
| Análise de documento estático | 7 dias | Documento não muda |
| Dados de produto (preços) | 5 min | Muda frequentemente |
| Resposta personalizada por usuário | Não cachear globalmente | Varia por contexto |
| Embedding de texto | Indefinido (até mudança) | Hash do texto como chave |

## Cache semântico

O cache semântico reutiliza respostas para perguntas **semanticamente similares** — mesmo quando o texto exato é diferente:

```python
# cache/semantic_cache.py
import numpy as np
from openai import AsyncOpenAI

class SemanticCache:
    def __init__(self, redis_client, embedding_client: AsyncOpenAI, threshold: float = 0.95):
        self._redis = redis_client
        self._openai = embedding_client
        self._threshold = threshold

    async def _embed(self, text: str) -> list[float]:
        response = await self._openai.embeddings.create(
            model="text-embedding-3-small",
            input=text,
        )
        return response.data[0].embedding

    def _cosine_similarity(self, a: list[float], b: list[float]) -> float:
        a_arr = np.array(a)
        b_arr = np.array(b)
        return float(np.dot(a_arr, b_arr) / (np.linalg.norm(a_arr) * np.linalg.norm(b_arr)))

    async def find_similar(self, query: str) -> dict | None:
        """
        Busca resposta cacheada com similaridade >= threshold.
        AVISO: implementação simplificada para fins didáticos.
        Em produção, use pgvector ou Redis Vector Search.
        """
        query_embed = await self._embed(query)

        # Itera sobre chaves (ineficiente — use pgvector em produção)
        async for key in self._redis.scan_iter("ai:semantic:*"):
            cached = await self._redis.hgetall(key)
            if not cached:
                continue
            cached_embed = json.loads(cached.get("embedding", "[]"))
            if not cached_embed:
                continue
            similarity = self._cosine_similarity(query_embed, cached_embed)
            if similarity >= self._threshold:
                return json.loads(cached["response"])
        return None
```

**Atenção:** cache semântico com busca por iteração é ineficiente em escala. Para produção, armazene embeddings no pgvector e use busca por similaridade indexada.

## Medindo o impacto

```python
import time
import asyncio

async def benchmark_cache(service: CachedAnalysisService, text: str, calls: int = 10):
    """Compara latência e custo com e sem cache para N chamadas idênticas."""
    results = []
    for i in range(calls):
        start = time.monotonic()
        result = await service.classify_text(text)
        latency = int((time.monotonic() - start) * 1000)
        results.append({
            "call": i + 1,
            "latency_ms": latency,
            "from_cache": result["from_cache"],
        })

    hits = [r for r in results if r["from_cache"]]
    misses = [r for r in results if not r["from_cache"]]

    print(f"Total: {calls} chamadas")
    print(f"Cache hits: {len(hits)} | Miss: {len(misses)}")
    print(f"Latência média (hit): {sum(r['latency_ms'] for r in hits) / max(len(hits), 1):.0f}ms")
    print(f"Latência média (miss): {sum(r['latency_ms'] for r in misses) / max(len(misses), 1):.0f}ms")
```

Resultado típico para 10 chamadas idênticas:

```
Total: 10 chamadas
Cache hits: 9 | Miss: 1
Latência média (hit): 3ms
Latência média (miss): 1.847ms
```

## Invalidação de cache

Três estratégias de invalidação:

1. **TTL:** o cache expira automaticamente. Simples, mas pode retornar dados desatualizados até o TTL expirar.
2. **Invalidação ativa:** quando os dados mudam, deletar a chave explicitamente.
3. **Cache bust por versão de prompt:** incluir a versão do prompt na chave — ao mudar o prompt, todas as chaves antigas ficam obsoletas automaticamente.

```python
# Invalidação ativa ao atualizar um documento
async def on_document_updated(document_id: str, redis_client) -> int:
    """Remove todos os caches relacionados ao documento atualizado."""
    pattern = f"ai:embed:{document_id}:*"
    keys_deleted = 0
    async for key in redis_client.scan_iter(pattern):
        await redis_client.delete(key)
        keys_deleted += 1
    return keys_deleted
```

## Riscos do cache em sistemas de IA

| Risco | Mitigação |
|---|---|
| Resposta desatualizada | TTL adequado ao dinamismo dos dados |
| Vazamento de resposta entre usuários | Incluir `user_id` na chave para respostas personalizadas |
| Cache de conteúdo impróprio | Validar saída antes de cachear |
| Custo de memória com TTL longo | Monitorar uso de memória Redis |

## Prática

**Exercício:** Adicione cache Redis a uma chamada de LLM de classificação de texto. Use hash SHA-256 do prompt como chave com TTL de 1 hora. Meça latência e custo para 10 chamadas idênticas.

**Passo a passo:**

1. Instale: `pip install redis[asyncio]`
2. Implemente `LLMResponseCache` com `get` e `set`
3. Implemente `CachedAnalysisService` que usa o cache
4. Execute `benchmark_cache` com 10 chamadas idênticas
5. Valide que a segunda chamada retorna `from_cache: true`
6. Verifique nos logs que apenas 1 chamada real foi feita à API

**Critérios de validação:**
- A segunda chamada com o mesmo prompt retorna do cache (não chama a API)
- A chave inclui a versão do prompt para invalidação controlada
- Chamadas cacheadas têm latência `<` 10ms
- Logs diferenciam `cache_hit` de `cache_miss`

- [ ] Respostas personalizadas por usuário incluem `user_id` na chave
